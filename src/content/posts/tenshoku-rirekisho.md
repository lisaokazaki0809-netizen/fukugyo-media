転職活動の最初の関門が書類選考。職務経歴書の書き方を調べてまとめた。

## 職務経歴書の基本構成

| セクション | 内容 | 目安の分量 |
|----------|------|----------|
| 職務要約 | 経歴のハイライトを3から5行で | 200から300文字 |
| 職務経歴 | 時系列で各社での業務内容と成果 | メイン部分 |
| スキル・資格 | 使えるツールや保有資格 | 箇条書き |
| 自己PR | 強みとそれを裏付けるエピソード | 300から500文字 |

全体でA4用紙2枚が理想。3枚以上になると読まれない可能性が高い。

## 20代と30代で求められることの違い

| 項目 | 20代の経歴書 | 30代の経歴書 |
|------|-----------|-----------|
| 評価軸 | ポテンシャル・やる気 | 実績・即戦力 |
| アピール内容 | 学ぶ姿勢、成長意欲 | 何ができるか、何を成し遂げたか |
| 数字 | なくても許される | 必須 |
| マネジメント | 不要 | あると大きなプラス |
| 転職理由 | 前向きならOK | 論理的な説明が求められる |

## 数字で語るテクニック

30代の職務経歴書で最も重要なのが「数字」。抽象的な表現を具体的な数字に変換するだけで、説得力が全く変わる。

### 変換例

| NGな表現 | OKな表現 |
|---------|---------|
| 営業成績が良かった | 営業目標を3期連続達成。最高は目標比135% |
| 業務効率化に貢献した | RPA導入により月40時間の工数を削減 |
| チームをまとめた | 5名のチームリーダーとして新規プロジェクトを推進、売上3,000万円を達成 |
| コスト削減に取り組んだ | 外注費の見直しにより年間200万円のコスト削減を実現 |
| 顧客満足度を向上させた | NPS（推奨度）を前年比15ポイント改善 |
| 新規事業に携わった | 新規サービスの企画から立ち上げまで担当。初年度売上5,000万円 |

### 数字がない場合の対処法

「数字で語れる実績がない」と思っている人でも、以下の視点で掘り起こせる。

| 視点 | 質問 | 数字化の例 |
|------|------|----------|
| 規模 | 担当した案件の予算は？ | 年間予算3億円のプロジェクトを管理 |
| 人数 | 関わった人は何人？ | 社内10部門、外部3社との調整を担当 |
| 期間 | どれくらいの期間で達成した？ | 通常6ヶ月かかる導入を3ヶ月で完了 |
| 改善率 | ビフォーアフターの差は？ | エラー率を5%から0.5%に改善 |
| 頻度 | どれくらいの頻度でやっていた？ | 月平均30件の問い合わせ対応 |

## よくあるNGパターン

### NG1: 業務内容の羅列だけ

「何をしたか」の列挙になっている経歴書が非常に多いらしい。採用担当者が知りたいのは「何を成し遂げたか」。

| NG | OK |
|----|-----|
| 営業活動全般を担当 | 法人向け新規営業を担当。年間50社へのアプローチで15社の新規契約を獲得 |
| 経理業務に従事 | 月次決算、四半期決算の主担当として3名のチームを統括。決算日を2日短縮 |

### NG2: 自己PRが抽象的

「コミュニケーション能力が高い」「責任感がある」は誰でも書ける。具体的なエピソードで示す必要がある。

| NG | OK |
|----|-----|
| コミュニケーション能力が高い | 部門間の利害が対立するプロジェクトで、定例ミーティングの設計と合意形成を主導。3ヶ月で全部門の合意を取り付けた |
| 責任感がある | 担当システムの障害発生時に、休日出勤で対応。原因特定から復旧まで4時間で完了し、影響を最小限に抑えた |

### NG3: 長すぎる

A4で3枚以上の経歴書は、忙しい採用担当者に読んでもらえない。要点を絞って2枚にまとめるのが鉄則。

### NG4: 誤字脱字

意外と多いらしい。「細部に気を配れない人」という印象を与えてしまう。必ず第三者にチェックしてもらうか、校正ツールを使う。

| チェック項目 | 確認 |
|------------|------|
| A4で2枚以内に収まっているか | |
| 各職歴に具体的な数字が入っているか | |

| 誤字脱字はないか（第三者にも確認） | |

## まとめ

職務経歴書は「自分のプレゼン資料」。30代は具体的な数字と成果で語ることが求められる。

自分もまず今までの仕事を棚卸しして数字に落とし込む作業から始める。転職しなくても、自分のキャリアを振り返る良い機会になる。
